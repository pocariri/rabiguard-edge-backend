# VLM + YOLO (NPU & CPU Hybrid Architecture)

이 디렉토리는 Hailo NPU(SCDepthV3, VLM)와 CPU(YOLO26n NCNN)를 혼합하여 실시간으로 객체를 감지하고, 깊이(Depth)를 검증한 뒤, 이상 상황을 VLM으로 분석하는 고도로 최적화된 아키텍처 구현체들을 포함하고 있습니다.

## 📂 주요 파일 구성

### 1. `arch1_headless.py` (최종 최적화 버전 🚀)
*   **방식**: 디스플레이 출력 없이(Fakesink) 백그라운드에서 실행되는 **Extreme Real-time** 파이프라인입니다.
*   **핵심 최적화 기술**:
    *   **Truly Headless**: `hailooverlay` 등 모든 시각화 요소를 제거하여 CPU 부하를 최소화했습니다.
    *   **지능적 프레임 드랍 (yolo_ready)**: YOLO 워커가 처리 중일 때 콜백에서 데이터 복사를 건너뛰어 파이프라인 FPS를 30 수준으로 유지합니다.
    *   **리소스 격리**: `OMP_NUM_THREADS=2` 설정을 통해 GStreamer 파이프라인을 위한 전용 CPU 코어를 확보하여 데이터 누락을 방지합니다.
    *   **좌표 복원 방식**: YOLO 추론은 `320x320`에서 수행하여 속도를 높이고, 결과 좌표를 원본 해상도(`640x480`)로 복원하여 정확한 Depth를 측정합니다.
*   **실시간 디버깅 모드**:
    *   `--visualize` 인자를 추가하여 실행하면 별도의 비동기 스레드에서 분석 상황(YOLO 박스, ROI)을 출력합니다.
    *   시각화 큐가 가득 차면 프레임을 즉시 버리는 방식을 채택하여 메인 추론 성능에 영향을 주지 않습니다.
*   **실행**: `python3 arch1_headless.py` (또는 `python3 arch1_headless.py --visualize`)

### 2. `arch1_headless_v2.py` (실험적 버전)
*   **특징**: NCNN 입력 해상도를 `192x192`까지 축소하는 등 극한의 추론 속도 향상을 위한 추가 실험이 적용된 버전입니다.

### 3. `arch1_yolo_depth.py` (실시간 시각화 버전)
*   **특징**: 카메라 프레임에 깊이 맵(Depth Map)을 오버레이하여 보여줍니다. 개발 및 디버깅 시 시각적 검증용으로 적합합니다.

### 4. `arch1_dynamic/` (동적 구역 설정 버전)
*   **특징**: Firestore 리스너를 통해 실시간으로 ROI(관심 구역) 좌표를 업데이트할 수 있는 확장 아키텍처입니다.

## ⚙️ 로직 흐름 (Headless 기준)

1.  **YOLO 탐지 (CPU)**: NCNN 기반 YOLO26n 모델로 사람(Class 0)을 실시간 감지합니다.
2.  **시간적 검증 (2s)**: 감지된 객체가 ROI 내에 최소 `2초` 이상 머무는지 확인하여 일시적인 노이즈를 필터링합니다.
3.  **입체적 검증 (Depth)**: SCDepthV3(NPU)가 생성한 깊이 맵을 활용, 객체의 깊이와 구역의 평균 깊이를 비교(`0.5m` 이내)하여 실제 구역 내 존재 여부를 확증합니다.
4.  **VLM 상황 분석 (NPU)**: 모든 조건이 충족되면 VLM(Qwen2-VL)이 해당 프레임을 분석하여 "무엇을 하고 있는지" 한 문장으로 요약 알림을 생성합니다.

## 🛠️ 기술적 해결 과제 (History)

*   **GStreamer 1.26.2 호환성**: `StructureWrapper` 버그를 우회하기 위한 내부 구조체 언래핑 로직(`get_caps_from_pad_fixed`)이 포함되어 있습니다.
*   **메모리 관리**: 파이썬 바인딩에서 지원하지 않는 `buffer.ref()` 대신 **조건부 데이터 복사** 전략을 채택하여 안정성을 확보했습니다.
*   **지연시간(Latency) 최소화**: 파이프라인 내 `bypass_max_size_buffers` 값을 조정하여 처리 지연을 최소화했습니다.
