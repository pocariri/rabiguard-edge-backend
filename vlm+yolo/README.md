# VLM + YOLO (NPU & CPU Hybrid Architecture)

이 디렉토리는 Hailo NPU(SCDepthV3, VLM)와 CPU(YOLO26n NCNN)를 혼합하여 실시간으로 객체를 감지하고, 깊이(Depth)를 검증한 뒤, 이상 상황을 VLM으로 분석하는 고도로 최적화된 아키텍처 구현체들을 포함하고 있습니다.

## 📂 주요 파일 구성

### 1. `arch1_headless.py` (최종 최적화 버전 🚀)
*   **방식**: 디스플레이 출력 없이(Fakesink) 백그라운드에서 실행되는 **Extreme Real-time** 파이프라인입니다.
*   **핵심 최적화 기술**:
    *   **Truly Headless**: `hailooverlay` 등 모든 시각화 요소를 제거하여 CPU 부하를 최소화했습니다.
    *   **지능적 프레임 드랍 (yolo_ready)**: YOLO 워커가 처리 중일 때 콜백에서 데이터 복사를 건너뛰어 파이프라인 FPS를 안정적으로 유지합니다.
    *   **리소스 격리**: `OMP_NUM_THREADS=2` 설정을 통해 GStreamer 파이프라인을 위한 전용 CPU 코어를확보하여 데이터 누락을 방지합니다.
*   **모니터링 및 디버깅**:
    *   **상세 로깅**: 객체 진입 시 시간적(Temporal) 대기 상태와 물리적(Physical) 거리 차이를 실시간 로그로 출력하여 판단 과정을 투명하게 보여줍니다.
    *   **정기 분석 스냅샷**: 10초마다 탐지된 객체 박스, ID, 구역 선이 포함된 스냅샷을 `_outputs/snapshots/`에 저장하여 시스템 상태를 시각적으로 상시 확인 가능합니다.
    *   **VLM 이벤트 저장**: VLM 분석이 수행되는 순간의 원본 프레임을 별도로 저장합니다.
*   **실행**: `python3 arch1_headless.py`

### 2. `dynamic_roi_extractor.py` (동적 구역 추출 도구 🔍)
*   **역할**: 카메라 프레임에서 세그멘테이션(YOLOE-seg)을 통해 감시 구역으로 설정 가능한 후보들을 자동으로 추출합니다.
*   **주요 기능**:
    *   **좌표계 매칭**: 내부적으로 `640x480` 리사이즈를 수행하여 추출된 다각형 좌표를 `arch1_headless.py`의 좌표계와 즉시 호환되도록 제공합니다.
    *   **배경 투명 처리**: 추출된 객체만 남기고 나머지는 투명하게 처리된 4채널(BGRA) 이미지를 생성하여 UI 활용도를 높였습니다.
    *   **다각형 단순화**: `cv2.approxPolyDP`를 통해 구역 좌표의 개수를 최적화하여 연산 효율을 높였습니다.

### 3. `arch1_dynamic/` (동적 구역 설정 아키텍처)
*   **특징**: Firestore 리스너를 통해 실시간으로 ROI(관심 구역) 좌표를 업데이트할 수 있는 확장 아키텍처입니다.

## ⚙️ 로직 흐름 (Headless 기준)

1.  **YOLO 탐지 (CPU)**: NCNN 기반 YOLO26n 모델로 사람(Class 0)을 실시간 감지합니다.
2.  **시간적 검증 (2s)**: 감지된 객체가 ROI 내에 최소 `2초` 이상 머무는지 확인하며, 진행 상황을 로그로 출력합니다.
3.  **입체적 검증 (Depth)**: SCDepthV3(NPU) 깊이 맵을 활용, 사람과 구역 바닥의 거리 차이(`0.5m` 이내)를 검증하여 실제 구역 내 존재 여부를 확증합니다.
4.  **VLM 상황 분석 (NPU)**: 모든 조건이 충족되면 VLM(Qwen2-VL)이 프레임을 분석하고 요약 알림을 생성하며, 분석에 사용된 이미지를 저장합니다.

## 🛠️ 기술적 해결 과제 (History)

*   **GStreamer 1.26.2 호환성**: `StructureWrapper` 버그를 우회하기 위한 내부 구조체 언래핑 로직(`get_caps_from_pad_fixed`)이 포함되어 있습니다.
*   **메모리 관리**: 파이썬 바인딩에서 지원하지 않는 `buffer.ref()` 대신 **조건부 데이터 복사** 전략을 채택하여 안정성을 확보했습니다.
*   **데이터 시각화**: Headless 환경에서도 시스템의 판단 근거를 확인하기 위해 **상세 로깅 및 정기 스냅샷 저장** 시스템을 구축했습니다.
